# 老版本 vs 副本版本 差异说明

## 文件对照

| 项目 | 老版本（无问题） | 当前副本版本 |
|------|------------------|--------------|
| 文件名 | `custom_driving_sync - v 15 seperate ver.py` | `可视化传输速率版本custom_driving_sync - v 15 seperate ver - 副本.py` |
| 约行数 | ~5330 行 | ~6970 行 |

---

## 1. 结构与依赖

- **老版本**：开头直接 `import tkinter`，无 Tcl/Tk 路径配置，无 matplotlib。
- **副本版本**：开头有 **venv 下 Tcl/Tk 路径自动配置**；**依赖 matplotlib**，用于 LSL 实时曲线（“可视化传输速率”）。

---

## 2. 界面与功能

- **老版本**：无 LSL 通道实时绘图，无“传输速率曲线”等可视化，界面更简单。
- **副本版本**：多出 **LSL 信号监视器**（matplotlib 曲线、定时刷新），以及更多状态/队列显示，主线程要处理的 UI 和定时器更多。

---

## 3. 驾驶数据与 UI 更新（和卡死相关）

| 行为 | 老版本 | 副本版本（我们改过之后） |
|------|--------|---------------------------|
| 收到 LSL 驾驶数据时 | 在 **LSL 线程里直接** 调用 `_display_driving_params()` | **不**在 LSL 线程里调；改为 **主线程定时器** 每 250ms 调 `_tick_driving_display` → `_display_driving_params()` |
| 写入驾驶缓存后 | 每次写入后 **直接** 调用 `_update_cache_status()`（约 150 次/秒） | 写入路径 **不再** 调 `_update_cache_status()`；仅由 **每 5 秒** 的 `_start_cache_monitoring` 和 **每 1 秒** 的 `_tick_lightweight_status` 更新状态 |
| 日志 `log()` | 每次 `log()` 都 `root.after(0, update_log)`，由主线程执行 | 改为 **日志入队 `_log_queue`**，主线程在 `_tick_driving_display` 里 **定时排出**（`_drain_log_queue`），不再为每条日志发 `after(0)` |

老版本：子线程会频繁往主线程塞 `after(0)`（显示、状态、日志）。  
副本版本：减少了这些 `after(0)`，改为“主线程定时做显示 + 状态 + 排日志”，但多了“可视化”的绘图和定时器。

---

## 4. 与 UXF 同源、状态显示（副本独有）

- **副本版本** 在状态栏增加了 **“驾驶数据: N 条 (来源:Unity LSL)”**，并增加了 **每 1 秒** 的轻量状态刷新（`_tick_lightweight_status`），让“驾驶数据条数”尽快更新。
- 老版本没有“来源: Unity LSL”的提示，也没有 1 秒一次的轻量状态刷新。

---

## 5. 为什么“老版本没问题”可能更稳

1. **没有 LSL 曲线**：少掉 matplotlib 的绘图和定时器，主线程负载小，即使子线程偶尔 `after(0)` 也不容易卡死。
2. **功能少**：少很多状态、队列、可视化相关代码，逻辑更简单，出问题的点更少。
3. **使用场景**：若老版本时 Unity 发流/标记没那么猛，或采集时间较短，`after(0)` 堆积不明显，所以“没问题”。

---

## 6. 若想兼顾“稳定”和“副本里的新功能”

- **要最大稳定、能接受没有曲线**：继续用 **老版本** 做正式采集。
- **要用副本的“可视化 + 与 UXF 同源提示”**：用 **副本版本**，并确保：
  - 已按当前代码使用 **日志队列 + 主线程排出**、
  - **驾驶显示 / 状态更新** 只在主线程定时器里做、
  - 若仍卡顿，可尝试 **关掉或降低 LSL 曲线刷新频率**（在副本里把 plot 的 `update_interval` 调大或暂时禁用绘图）。

---

## 7. 小结表

| 项目 | 老版本 | 副本版本 |
|------|--------|----------|
| Tcl/Tk 配置 | 无 | 有（venv） |
| matplotlib / LSL 曲线 | 无 | 有 |
| 驾驶 UI 更新方式 | 子线程调 `_display_driving_params` | 主线程定时器 |
| 状态栏更新方式 | 写缓存后即 `_update_cache_status` | 定时 1s/5s 更新 |
| 日志更新方式 | 每次 `after(0)` | 队列 + 主线程排出 |
| “来源: Unity LSL” | 无 | 有 |
| 轻量状态 1 秒刷新 | 无 | 有 |
