# 多模态数据采集软件开源整理检查清单

## 项目结构检查
- [ ] 项目根目录包含标准的开源项目文件结构
- [ ] 存在README.md文件且内容完整
- [ ] 存在LICENSE文件且许可证合适
- [ ] 存在.gitignore文件配置正确
- [ ] 存在requirements.txt或pyproject.toml文件
- [ ] 存在src/或类似源代码目录
- [ ] 存在docs/或类似文档目录
- [ ] 存在examples/或类似示例目录

## 代码质量检查
- [ ] 主要Python文件结构清晰，命名规范
- [ ] 代码包含适当的注释和文档字符串
- [ ] 没有硬编码的敏感信息（API密钥、密码等）
- [ ] 代码文件包含版权头信息
- [ ] 导入语句组织良好，无循环依赖
- [ ] 错误处理机制完善

## 文档完整性检查
- [ ] README.md包含项目简介和主要功能
- [ ] 安装说明详细且可执行
- [ ] 使用指南清晰易懂
- [ ] API文档完整（如适用）
- [ ] 贡献指南存在且合理
- [ ] 联系信息和维护者信息完整

## 版权合规检查
- [ ] 所有代码文件都有适当的许可证声明
- [ ] 第三方依赖的许可证兼容
- [ ] 没有侵犯第三方版权的代码
- [ ] 必要的归属和致谢已添加
- [ ] 商标和品牌使用合规

## 数据清理检查
- [ ] 实验数据和个人信息已删除
- [ ] 临时文件和缓存已清理
- [ ] 日志文件不包含敏感信息
- [ ] 配置文件模板化，无实际配置数据

## GitHub仓库检查
- [ ] GitHub仓库已创建并配置
- [ ] 项目描述准确完整
- [ ] 标签和主题设置合适
- [ ] 初始提交已推送到远程
- [ ] 仓库设置为公开（如计划开源）
- [ ] 问题模板和贡献指南已配置

## 功能验证检查
- [ ] 核心功能可以正常运行
- [ ] 依赖安装过程无误
- [ ] 示例代码可以执行
- [ ] 配置文件可以正确加载
- [ ] 错误处理机制有效

## 发布准备检查
- [ ] 版本号已确定并标记
- [ ] 发布说明已准备
- [ ] 项目状态标记为稳定或测试版
- [ ] 联系方式和反馈渠道已提供
- [ ] 后续维护计划已制定
